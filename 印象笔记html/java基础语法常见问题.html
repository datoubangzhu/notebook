2.基本数据类型问题：
float a = 1.1;可以吗，不可以，因为浮点默认为double类型，可以写成float = 1.1f；或者强转写成 float a = (float) 1.1;
long  j = 1234567891011;可以吗，不可以，因为整型默认为int，但是长度超过了int范围，可以写成long j = 1234567891011L；
float a = 1;可以吗，可以，int可以隐式转为float；
隐式转换的方向（不报错，但是会损失精度）：
byte,short,char  => int =>long =>float=>double  
(1)  char字符为什么能转换为int类型：遵从ASCII表进行转换；
(2) long为什么能隐式转成float：转型会损失精度。包括int转为float等等都是的。    
(3)short i = 1; 为什么能编译通过：
将一个整型的常量（字面值为int）赋给一个byte或者short类型的变量，只要不超过byte或short取值范围，就可以通过编译
将一个浮点型的常量（字面值的double）赋给一个float类型的变量，即使不超过float的取值范围，也会出现编译错误

2.try catch finally 中 有return  break  continue 时的执行顺序（未完）
(1) try 或者catch 中有return时，try或者catch中的代码执行到return结束，最后finally都会正常执行，finally中有return则执行到return前
(2)关于try、catch、finally中return 值对结果的影响：
return
  • 对于返回值是基本数据类型的来说，return相当于将返回值入操作数栈，那么这个值就是方法要的返回值，后续再对变量进行任何操作，都不会改变栈中的值，除非再次给栈赋值，就是catch或者finally里再有return，但是java不推荐这么做，finally里写return会有警告提示。
     以下博客内容一般，但是举得的例子可以作为验证https://blog.csdn.net/qq_36262896/article/details/79779953
  • 对于引用数据类型，结果直接指向地址，return之后还是地址，那么例如try里return之后，在catch里或者finally里还有对引用数据类型的操作，则会继续影响最终的结果，无关有没有return。还可以拿上面博客作为demo验证



2.break，continue使用（continue是loop的“专利”，注意switch不可以写continue）
  • if中写break会报错，break只用在switch或者loop循环中；在switch中，break用于中断switch执行，不写break则意味着参数可以继续判断是否满足其他条件，写break则跳出switch剩余判断，继续执行其他代码（switch花括号之外的）；

  • if和switch中写continue会报错，continue只用在loop循环中；

在循环中，break用于中断这层所有循环（只能跳出一层），continue用于跳出此次循环（continue后的代码不再执行），直接开始下一个循环

3.java中引用传递还是值传递：
对于基本数据类型参数，是值传递，等于是copy一份值过去，其运算修改操作对原来的值没有任何影响。

对于对象类型，也是值传递，传递的是地址的引用的copy值，等于复制了一份指向同样地址的引用，那么无论是谁对于对象属性进行操作，对原来对象的属性都有影响，对于copy过来的引用重新赋值新的引用，改变的是备份值的引用，原先指向内存空间的被copy引用依然存在。下面的代码说明了这个问题：
public static void methodTwo(List<Integer> a) {
        a = new ArrayList<Integer>();// 就在这加了这一句
        a.add(101);
}
    
    public static void main(String[] args) {
        List<Integer> a = new ArrayList<Integer>();
        a.add(100);
        methodTwo(a);
        System.out.println("main a="+a.toString());
    }
注意输出结果是:   main a = [100]

15.为何重写equals方法就必须要重写hashCode方法。
因为对于Object，不同对象地址的对象，最后计算得到的hashCode可能是相同的，这时候再根据Object.equal()方法比较地址值，就可以确定唯一。
当equal被重写，就不具备与Object的hashCode一同计算唯一元素的作用了。所以重写了equals方法，hashCode也要相应改变。
参考博客：https://www.jianshu.com/p/f9cc84047bd6


16.final变量可以在定义时不赋初值吗？静态成员变量可以在定义时不赋初值吗？
非静态成员变量用final定义必须要再创建对象之前赋初值：直接定义时赋初值或者在构造函数里赋初值；
静态成员变量用final定义必须在定义时赋初值。而static定义的变量可以在定义时不赋值。
总之一句话：final修饰的，创建对象之前一定要赋初值（如果有static修饰那只必须要在定义时即赋初值）
